## Завдання 1

Потрібно реалізувати систему для касового апарата, яка визначає спосіб видачі решти покупцеві.

Набір монет:

```python
[50, 25, 10, 5, 2, 1]
```

Реалізовано дві функції з назвами, зазначеними в умові:

```python
find_coins_greedy(amount)
```

```python
find_min_coins(amount)
```

### Функція `find_coins_greedy`

Функція використовує жадібний алгоритм.

Алгоритм спочатку вибирає найбільші доступні номінали монет, а потім переходить до менших.

Для суми `113` результат:

```python
{50: 2, 10: 1, 2: 1, 1: 1}
```

Часова складність:

```text
O(n)
```

де `n` — кількість номіналів монет.

Оскільки у цьому завданні кількість номіналів стала (`6`), жадібний алгоритм працює дуже швидко навіть для великих сум.

### Функція `find_min_coins`

Функція використовує динамічне програмування.

Алгоритм знаходить мінімальну кількість монет, необхідних для формування кожної суми від `1` до заданої суми `amount`.

Для суми `113` результат:

```python
{1: 1, 2: 1, 10: 1, 50: 2}
```

Це той самий набір монет, що й у жадібному алгоритмі, але порядок ключів у словнику може відрізнятися.

Часова складність:

```text
O(amount * n)
```

де `amount` — задана сума, `n` — кількість номіналів монет.

Просторова складність:

```text
O(amount)
```

### Порівняння ефективності

У файлі `task_1.py` виконується порівняння часу роботи двох алгоритмів для сум:

```python
[113, 1000, 10000, 100000]
```

Жадібний алгоритм має перевагу для великих сум, тому що проходить лише по списку номіналів монет. Його швидкість майже не залежить від величини суми.

Алгоритм динамічного програмування є універсальнішим. Він перевіряє всі проміжні суми та може знайти мінімальну кількість монет навіть для таких наборів номіналів, де жадібний алгоритм не гарантує оптимального результату. Але для великих сум він працює повільніше та використовує більше пам'яті.

Висновок: для заданого набору монет `[50, 25, 10, 5, 2, 1]` жадібний алгоритм є швидшим і практичнішим, а динамічне програмування є більш універсальним підходом.

## Завдання 2

Потрібно обчислити значення визначеного інтеграла методом Монте-Карло.

Для прикладу використано функцію:

```python
f(x) = x ** 2
```

Межі інтегрування:

```python
a = 0
b = 2
```

Очікуване аналітичне значення інтеграла:

```python
8 / 3 = 2.6666666666666665
```

### Алгоритм Монте-Карло

1. Створюється прямокутник, який містить область під графіком функції.
2. У прямокутнику генеруються випадкові точки.
3. Підраховується кількість точок, які потрапили під графік функції.
4. Площа прямокутника множиться на частку таких точок.
5. Отримане значення порівнюється з результатом `scipy.integrate.quad` та аналітичним обчисленням.

### Порівняння результатів

У файлі `task_2.py` метод Монте-Карло запускається для різної кількості випадкових точок:

```python
[100, 1000, 10000, 100000]
```

Так можна побачити, як змінюється точність методу при збільшенні кількості точок.

Результат методу Монте-Карло є наближеним. Він близький до результату, який повертає функція `quad` з підмодуля `integrate` бібліотеки SciPy.

Невелика різниця між результатами є нормальною, тому що метод Монте-Карло використовує випадкову генерацію точок. Якщо збільшити кількість випадкових точок, результат зазвичай стає точнішим, хоча через випадковість можливі невеликі коливання.

Після запуску `task_2.py` створюються два графіки:

```text

monte_carlo_convergence.png
```
